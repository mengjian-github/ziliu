## 1. 现有 Prompt 全景扫描

### 涉及文件

| 文件路径 | 功能 | 覆盖平台 |
|----------|------|----------|
| `src/app/api/short-text/generate/route.ts` | 短图文文案生成 | 小绿书、小红书图文、微博、即刻、X |
| `src/app/api/video/generate-metadata/route.ts` | 视频发布元数据 | 视频号、抖音、B站、小红书视频、YouTube |
| `src/app/api/video/convert-speech/route.ts` | 口播稿转换 | 视频号、抖音、B站、小红书视频、YouTube |

### 现有 Prompt 优缺点总览

**做得好的部分 ✅**
- 每个平台有独立 prompt，区分度明确
- 输出格式约束清晰（JSON/固定格式）
- 已包含 PREP 结构引导（口播稿）
- 小红书标点限制已考虑（≤1个问号/感叹号）
- 封面生成有详细的分类模板体系

**需要改进的部分 ❌**
- prompt 缺乏平台算法信号引导（如完播率、CES评分）
- 缺少反面示例/负面约束（"不要XX"）
- 标签策略缺乏「产品词+场景词+痛点词」矩阵指导
- 口播稿缺少「前3秒钩子」的强制约束
- 微博/即刻 prompt 过于简单，缺少风格锚点
- 未考虑AI内容检测规避（各平台都在打击纯AI内容）

## 2. 短图文生成 Prompt 优化

### 2.1 小红书图文笔记（`xiaohongshu_note`）

**现有 Prompt 问题分析**

```
现有：标题：6-20个汉字，偏爆款风格
问题：范围太宽，没有给出具体的爆款公式
```

```
现有：正文：200-900字，允许换行与emoji
问题：没有 emoji 使用频率指导，没有信息密度要求
```

```
现有：适当引导互动（如"评论区聊聊/你也遇到过吗"）
问题：互动引导太泛，没有引导产生高权重互动（≥8字评论）
```

**优化后 Prompt**

```typescript
xiaohongshu_note: `
你是"小红书图文笔记"运营助手，精通小红书 CES 评分算法。请把原始内容改写为高互动率的小红书笔记。

⚠️ 算法提示：小红书 CES 评分中关注=8分、评论/转发=4分、点赞/收藏=1分。
字数≥8字的场景化评论权重是普通评论的3倍，你的文案应当引导用户写出具体场景评论。

要求：
1) 标题：必须 18-20 个汉字（填满但不超）
   - 使用以下公式之一：
     a. 争议型："月薪3000买大牌？我靠这3招省出旅游基金"
     b. 场景型："早八通勤妆｜5分钟搞定的伪素颜小心机"  
     c. 数字型："亲测XX天｜从XX到XX的真实变化"
   - 最多1个感叹号或问号，禁止多重标点
   - 前8字必须包含核心关键词

2) 正文：300-700字
   - 结构：痛点引入（50字内）→ 解决方案（3个要点）→ 互动引导
   - 每2-3行插入1个emoji（不要连续emoji，自然穿插）
   - 每100字至少1个有效信息点（数据/步骤/对比/效果）
   - 引导高权重评论：用二选一提问（如"你是A还是B？评论区聊聊"）而非"你觉得呢"
   - 不要出现 Markdown 语法；不要输出图片URL

3) 话题标签：6-8个，按「产品词+场景词+痛点词+热门词」结构
   示例：["敏感肌", "换季护肤", "屏障修复", "油皮急救", "护肤干货", "好物分享"]

4) 可以根据配图信息自然地写"图1/图2…"（不要输出URL）

❌ 禁止：学术语气、长句式、"综上所述"类AI套话、过度emoji堆砌

输出必须是严格 JSON：
{"title":"...","content":"...","tags":["..."]}`,
```

**关键改进点**
| 维度 | 旧 | 新 |
|------|----|----|
| 标题字数 | 6-20（范围过宽） | 18-20（填满以获得最大展示面积） |
| 标题公式 | 仅"偏爆款" | 提供3种具体公式+示例 |
| emoji指导 | "允许emoji" | "每2-3行1个，自然穿插" |
| 互动引导 | "评论区聊聊" | 引导二选一提问以获得≥8字评论 |
| 信息密度 | 无 | "每100字≥1个有效信息点" |
| 标签策略 | "5-10个" | "6-8个，按矩阵结构" |
| 算法意识 | 无 | 加入CES评分提示 |

### 2.2 微信小绿书（`wechat_xiaolushu`）

**优化后 Prompt**

```typescript
wechat_xiaolushu: `
你是"微信小绿书（公众号图片消息）"运营助手。请把原始内容改写为适合在微信生态传播的小绿书文案。

⚠️ 平台特点：小绿书展示比例3:4，纯文字排版，用户在消息列表和常读列表中看到。
重点优化：标题吸引力 + 前3行抓住注意力 + 适合转发给朋友。

风格要求：类似小红书但更克制，纯文字，不使用Emoji或任何符号装饰。

要求：
1) 标题：8-18个汉字，实用导向（"X个方法""亲测有效的XX"），不要出现"标题："前缀
2) 正文：300-800字
   - 前3行（约50字）必须抓住核心痛点或抛出反常识观点
   - 以段落为主，每段3-5行
   - 纯文本，不要 Markdown/图片URL/长链接
   - 结尾可自然引导："更多内容可以关注我"
3) 话题标签：3-5个（不要带#号）
4) 可以根据配图信息自然地写"图1/第1张图里…"

❌ 禁止：Emoji、特殊符号装饰、标题党夸张、外链

输出必须是严格 JSON：
{"title":"...","content":"...","tags":["..."]}`,
```

**关键改进**
- 增加了「前3行抓住注意力」的约束（对应算法的点击率优化）
- 标题范围从6-20缩窄到8-18（更实用）
- 增加了段落结构指导
- 增加了引流引导（自然方式）

### 2.3 微博（`weibo`）

**现有问题**
- Prompt 过于简单，只有3条要求
- 缺少微博特有的表达风格指导
- 没有热点关联引导

**优化后 Prompt**

```typescript
weibo: `
你是"微博"运营助手。请把原始内容改写为适合微博信息流的短图文文案。

⚠️ 微博特点：信息密度极高、碎片化阅读、金句传播力强。
2025年微博更注重"活人感"和原创观点，同质化/低质内容会被限制推荐。

要求：
1) 正文：80-180字
   - 第一句必须是核心观点/金句/信息点（微博展开前只显示前几行）
   - 信息密度高：一句话一个信息点，去掉所有连接词和废话
   - 自然分2-3段（不要硬拆成一句一段）
   - 可以有态度/观点，不要中立到乏味
   - 适量使用emoji（0-3个，点缀而非堆砌）

2) 话题标签：1-3个（不要带#号）
   - 至少1个与当前内容直接相关的话题
   - 可包含1个泛领域话题用于蹭流量

3) 不要出现 Markdown 语法、图片URL、硬广

❌ 禁止：长句、学术表达、AI套话、过于官方的语气

输出必须是严格 JSON：
{"content":"...","tags":["..."]}`,
```

**关键改进**
- 增加「第一句必须是核心观点」（微博展开逻辑）
- 增加「活人感」风格指导（对应2025新规）
- emoji使用从完全未提及到给出具体范围
- 增加观点/态度引导

### 2.4 即刻（`jike`）

**现有问题**
- 语气要求只说了"真诚、有个人视角"，缺少具体锚点
- 没有引导「第一人称叙述」和「洞察感」

**优化后 Prompt**

```typescript
jike: `
你是"即刻"运营助手。请把原始内容改写为适合即刻发布的个人动态。

⚠️ 即刻社区文化：真实>套路，洞察>信息量，个人视角>客观报道。
即刻用户以互联网/科技/创业/产品从业者为主，信息素养高，极度反感营销腔。

要求：
1) 正文：120-280字
   - 必须用第一人称叙述（"我发现""我试了""最近在思考"）
   - 开头直入主题，不要铺垫（不要"今天想和大家分享一下"）
   - 有一个明确的个人洞察或观点，而非单纯信息罗列
   - 以段落为主，自然表达，能合并就不要拆段
   - 结尾可以抛出一个真诚的开放问题引发讨论
   
2) 话题标签：1-2个（不要带#号）
   - 选择精准的即刻圈子名称（如"独立开发者""产品沉思录""AI探索"）

3) 不要出现 Markdown 语法、图片URL

❌ 严禁：标题党、营销腔、"干货分享""建议收藏"、过度emoji、段落过多（不超过4段）

输出必须是严格 JSON：
{"content":"...","tags":["..."]}`,
```

**关键改进**
- 加入即刻社区文化描述（让AI理解平台调性）
- 明确「第一人称」「个人洞察」要求
- 加入反面约束（"不要铺垫""严禁营销腔"）
- 标签改为推荐即刻圈子名称

## 3. 视频元数据 Prompt 优化

### 3.1 抖音（`douyin`）

**现有问题**
- 标题要求"制造悬念或冲突"太泛，缺乏具体公式
- 描述30-55字没有提及「前3秒」对应的信息
- 标签没有引导混合使用大流量+垂直标签

**优化后 Prompt**

```typescript
douyin: `
请为以下文章内容生成适合抖音的发布元数据。

⚠️ 抖音算法核心：完播率 + 播放时长。前3秒决定用户是否继续观看。
2025年新算法更注重播放时长，标题和描述必须创造「打开+看完」的双重动力。

要求：
1. 标题（10-25字）：
   - 必须使用以下手法之一：悬念感（"结果让我震惊"）、数字冲击（"3天学会""90%的人不知道"）、反常识（"你一直做错了"）
   - 前5字必须抓住注意力
   - 示例好标题："这个功能90%的人都用错了""3天从小白到高手的秘诀"

2. 描述（30-50字）：
   - 节奏感强，短句为主
   - 包含1-2个emoji
   - 描述中不要带#话题标签
   - 必须包含1个核心钩子（悬念/利益点/共鸣点）

3. 标签（5-8个，#号格式）：
   - 前2个：泛流量标签（如#涨知识 #干货）
   - 中间3个：垂直领域标签
   - 后2个：长尾/场景标签
   
4. 封面建议：强视觉冲击力、高对比度、大字标题（6-9字），避免人物

5. 不能直接复述文章第一句话

请按以下格式输出：
标题：[标题内容]
描述：[描述内容]
标签：#标签1 #标签2 #标签3 #标签4 #标签5
封面：[封面建议]`,
```

### 3.2 B站（`bilibili`）

**现有问题**
- 标题只说"信息量大，可以稍长"，没给范围
- 没有引导使用【】标注系列/分类（B站文化）
- 未提及分区建议

**优化后 Prompt（关键变更部分）**

```
1. 标题（15-40字）：
   - 信息量大，体现专业性和内容量
   - 可使用【】标注系列/分类（如"【全面解析】""【实战教程】"）
   - 包含核心关键词用于搜索
   - 示例："【2025前端指南】React vs Vue vs Svelte 全方位实战对比"

2. 描述（150-250字）：
   - 详细介绍内容大纲，包含 2-3 个要点（用短句或项目符号）
   - 可包含时间戳占位符（如 "00:00 开头 / 02:30 核心对比"）
   - 描述中不要包含#话题标签
   - 结尾可加"一键三连支持一下～"

3. 标签：
   - 第1个：B站分区标签（如 #知识分享 #科技 #编程）
   - 其余2-4个：内容关键词标签

4. 额外输出一行：
   分区建议：[推荐的B站投稿分区]
```

### 3.3 小红书视频（`xiaohongshu`）

**现有问题**
- 描述要求200-500字太长，小红书视频描述通常更短
- 没有引导「前3秒」内容设计
- 未强调CES评分中互动引导的重要性

**优化后关键变更**

```
1. 标题：18-20字（填满展示空间）
   - 必须有场景感或体验感
   - 最多1个问号或感叹号

2. 描述：150-350字
   - 前2行（约30字）必须是最吸引人的信息（折叠前可见）
   - 每3-4行插入1个emoji
   - 结尾必须有互动引导，用二选一提问引导用户写≥8字评论
   - 描述正文中禁止#话题标签

3. 标签（6-8个）：
   - 按「品类词+功效词+人群词+场景词+热门词」结构排列
```

### 3.4 视频号（`video_wechat`）

**现有问题**
- 描述80-120字没有强调「社交传播力」
- 缺少微信生态联动引导（公众号引流等）

**优化后关键变更**

```
⚠️ 视频号核心：社交推荐占50%权重。好友点赞/分享会直接推给其好友。
内容必须让用户「看完想转发给朋友」。

2. 描述（80-120字）：
   - 温和友好，引导互动和分享
   - 突出「对谁有用」（适合XX的朋友看看）
   - 描述中严禁#话题标签
   - 可自然引导："觉得有用就转发给需要的朋友"（利用社交推荐机制）
```

## 4. 口播稿转换 Prompt 优化

### 4.1 抖音口播稿

**现有问题**
- 只说了"开头要有冲击力"，没给具体的钩子写法
- 没有强调「前3秒」是生死线（>45%跳出率直接限流）
- 缺少具体的钩子词汇库

**优化后 Prompt**

```typescript
douyin: `
请将以下文章内容转换为适合抖音的口播稿：

⚠️ 核心铁律：前3秒跳出率>45%会直接限流。前15字决定生死。

要求：
1. 时长控制在30秒-1分钟（约100-200字）

2. 前3秒（前15-20字）必须是强钩子，使用以下模式之一：
   - 反常识："你一直在用错这个功能"
   - 痛点直击："为什么你学了XX还是不会"
   - 悬念："看到最后你会感谢我"
   - 利益承诺："学会这招直接省XX"
   
3. 语调活泼有节奏感：
   - 短句为主（每句≤15字）
   - 每句话一个信息点
   - 适当使用反问和设问制造节奏

4. 使用 PREP 结构：
   - Point：核心观点（即前3秒钩子）
   - Reason：为什么（1-2句）
   - Example：怎么做/举例（核心部分）
   - Point：总结 + 强引导（"关注我，教你更多XX"）

5. 最后一句必须是关注引导

请直接输出口播稿内容，不要其他说明。`,
```

### 4.2 B站口播稿

**现有问题**
- 虽说"可以相对复杂和深入"，但缺少具体的叙事节奏指导
- 没有提及弹幕互动引导（B站独特文化）

**优化后关键增补**

```
额外要求：
6. 每3-5分钟设置一个弹幕互动点：
   - "大家弹幕打出你的答案"
   - "猜猜看接下来会怎样"
   
7. 叙事节奏：
   - 前30秒：交代背景+核心问题（不要铺垫过长）
   - 中间：分3-5个论点，每个论点有起承转合
   - 结尾：总结 → 下期预告 → "一键三连支持一下！"

8. 可以适当使用B站梗/文化用语（"我直接一个好家伙""泪目了"）
```

### 4.3 视频号口播稿

**现有问题**
- 缺少社交传播导向的写作提示
- 没有引导「分享价值」的表达

**优化后关键增补**

```
额外要求：
6. 内容必须让观众觉得"应该转发给朋友/家人"
   - 可以直接说："如果你身边也有XX的朋友，帮我转发给他"
   - 突出实用价值和适用人群

7. 语气特点：像一个靠谱的朋友在分享经验，不是在做演讲
```

### 4.4 小红书视频口播稿

**现有问题**
- "真实亲切，像分享给闺蜜"是对的但不够具体
- 缺少对视频完播率的考虑（前3秒）

**优化后关键增补**

```
额外要求：
6. 前3秒必须直接呈现结果或痛点（不要"大家好我是XX"开头）
   示例开头："用了这个方法之后我的XX直接好了80%"

7. 每15秒植入1个新信息点/转折，保持观看动力

8. 多用以下口语词缀增加真实感：
   "说真的""亲测""我当时直接""你们懂那种感觉吗"
```

## 5. 封面生成 Prompt 优化

### 现有封面系统评价

现有的 `buildCoverImagePrompt` 和分类模板系统（`coverTemplatesByCategory`）已经相当完善，包含了：
- 按平台区分规格和风格 ✅
- 按内容类别（tutorial/review/list等）匹配模板 ✅

## 6. 通用架构层面建议

### 6.1 增加「AI内容检测规避」机制

**问题**：2025-2026年各平台都在打击纯AI生成内容。小红书要求AI内容需修改≥30%并标注，公众号AI识别准确率达92%。

**建议**：在所有 prompt 中增加以下通用约束：

```
通用反AI检测指令（可作为 system prompt 前缀）：
---
请确保输出具有自然的人类写作特征：
1. 避免以下AI典型表达："综上所述""不难发现""众所周知""值得一提的是""总而言之"
2. 句式长短混合（不要每句话结构相似）
3. 适当使用不完整句或口语表达
4. 可以有小的主观偏好和个人化措辞
5. 避免过于工整的并列结构（不要每段都是"首先...其次...最后..."）
---
```

### 6.2 标签生成策略系统化

**现有问题**：各平台标签要求散落在各个 prompt 中，缺乏统一的标签策略矩阵。

**建议**：抽离一个标签生成子模块，按平台适配策略：

```typescript
const TAG_STRATEGIES: Record<string, string> = {
  xiaohongshu: '按「品类词+场景词+痛点词+人群词+热门词」结构，6-8个',
  douyin: '前2个泛流量标签+中间3个垂直标签+后2个长尾标签，5-8个',
  bilibili: '第1个为分区标签+其余为内容关键词，3-5个',
  weibo: '1个直接相关话题+1个泛领域话题，1-3个',
  jike: '精准的即刻圈子名称，1-2个',
  video_wechat: '精准话题+地理位置相关，3-5个',
};
```

### 6.3 温度参数（temperature）分平台优化

**现有情况**：所有平台统一使用 `temperature: 0.8`

**建议**：

| 平台/场景 | 建议温度 | 理由 |
|-----------|---------|------|
| 抖音标题/描述 | 0.9 | 需要创意性和冲击力 |
| 小红书笔记 | 0.85 | 需要创意但也要可控 |
| 知乎/掘金 | 0.6 | 专业内容需要准确严谨 |
| B站描述 | 0.7 | 信息量导向 |
| 微博短文 | 0.85 | 需要金句感 |
| 即刻动态 | 0.75 | 真诚自然，不要太花哨 |
| 视频号 | 0.7 | 温和友好，不要过于激进 |
| 口播稿（通用） | 0.7 | 需要流畅但不能太跳脱 |

### 6.4 内容长度智能适配

**现有问题**：长文章和短文章使用同一套 prompt，长文章可能被截断（`plainText` 传入量不限）。

**建议**：

```typescript
// 根据原文长度动态调整 prompt 策略
function adaptPromptByLength(plainText: string, platform: ShortTextPlatform): string {
  const length = plainText.length;
  if (length < 500) {
    return '原文较短，请在保持核心信息的同时适当扩展，增加场景化描述和个人化表达。';
  } else if (length > 3000) {
    return '原文较长，请提炼最核心的1-2个价值点进行改写，不要试图覆盖所有内容。';
  }
  return '请在保持核心信息的同时，进行风格化改写。';
}
```

### 6.5 增加 `system` 消息与 `user` 消息分离

**现有问题**：所有指令都放在单个 `user` 消息中。

**建议**：将平台风格定义放入 `system` 消息，将具体内容和任务指令放入 `user` 消息。这样：
- AI对角色定位更清晰
- 内容与指令分离，减少混淆
- 便于统一管理通用约束（如反AI检测指令）

```typescript
messages: [
  {
    role: 'system',
    content: `${ANTI_AI_PREFIX}\n${PLATFORM_SYSTEM_PROMPTS[platform]}`
  },
  {
    role: 'user', 
    content: `原始标题：${title}\n\n原始正文：${plainText}\n\n配图信息：${imagesHint}\n\n请按要求生成。`
  }
]
```

### 6.6 增加 Few-shot 示例

**现有问题**：prompt 中缺少输出示例，AI有时候格式不稳定。

**建议**：为每个平台增加1个简短的输出示例：

```
以下是一个优秀输出示例（仅供参考格式和风格，不要照抄内容）：
{"title":"换季烂脸急救指南｜敏感肌亲测3天见效","content":"每到换季脸就开始泛红脱皮 😢\n\n作为十年敏感肌，今年终于找到了救命方法...\n\n1️⃣ 精简护肤步骤...\n\n你们换季最怕什么？是泛红还是脱皮？评论区聊聊～","tags":["敏感肌","换季护肤","屏障修复","护肤急救","好物分享","烂脸急救"]}
```
